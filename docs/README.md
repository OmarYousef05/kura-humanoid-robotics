# Technical documentation

This folder is intended for public-safe technical case studies, diagrams, and engineering notes that expand on the main portfolio README.

Appropriate future documents include:

- Conceptual behaviour diagrams
- Testing methodology and lessons learned
- Competition retrospectives based on confirmed information
- Original engineering notes about ROS, humanoid robotics, and field debugging
- Public-safe explanations of adjust, chase, positioning, and ball-tracking work

Documentation must stay at a level that protects KURA's private implementation. It must not expose the full team architecture, complete behaviour trees, private source code, internal logs, robot configuration, competition credentials, restricted research, or material that has not been approved for publication.

Any description of personal work should distinguish confirmed individual contributions from the work of the wider team. KURA names, logos, robot designs, photographs, and team materials remain the property of their respective owners.
